# Development Guide\n\n## Project Architecture\n\n### Backend (Flask)\nThe backend is built with Flask and provides RESTful API endpoints for:\n- User authentication (JWT-based)\n- News analysis using ML models\n- Analysis history and feedback\n- Rate limiting and caching\n\n**Key Files:**\n- `app/__init__.py` - Flask app factory and initialization\n- `app/routes.py` - API endpoints using Flask-RESTX\n- `app/models.py` - SQLAlchemy database models\n- `app/ml_model.py` - Fake news detection model\n- `app/config.py` - Configuration management\n- `run.py` - Application entry point\n\n### Frontend (React + TypeScript)\nThe frontend is built with React and Vite, providing a modern SPA interface.\n\n**Key Technologies:**\n- React 18+ with TypeScript\n- Vite for fast development and building\n- Tailwind CSS for styling\n- Shadcn/UI for component library\n\n**Key Files:**\n- `frontend/src/App.tsx` - Main app component\n- `frontend/src/pages/` - Page components (Auth, Dashboard, etc)\n- `frontend/src/contexts/AuthContext.tsx` - Auth state management\n- `frontend/src/lib/api.ts` - API helpers and request utilities\n\n### Database\n- SQLite for development (configured in `app/config.py`)\n- Models: User, Analysis\n\n## Development Setup\n\n### 1. Virtual Environment\n```powershell\npython -m venv .venv\n.venv\\Scripts\\Activate.ps1\n```\n\n### 2. Install Dependencies\n```powershell\npip install -r requirements.txt\n```\n\n### 3. Run Development Server\n```powershell\npython run.py\n```\n\n### 4. Frontend Development (Optional)\nFor live reload of React code:\n```powershell\ncd frontend\nnpm run dev\n```\nThis starts Vite dev server at http://localhost:5173\n\n## Project Structure Detail\n\n### Backend Files\n\n#### app/__init__.py\n- Creates Flask app instance\n- Initializes extensions (SQLAlchemy, JWT, Cache, etc)\n- Sets up static file serving for React\n- Registers blueprints\n\n#### app/routes.py\n- `/api/auth/` - Authentication endpoints\n- `/api/analysis/` - Analysis endpoints\n- Uses Flask-RESTX for API documentation\n\n#### app/models.py\n- User model (username, email, password_hash)\n- Analysis model (predictions, confidence, timestamps)\n\n#### app/ml_model.py\n- SimpleFakeNewsDetector class\n- Uses heuristics for fake news detection\n- Analyzes text, URLs, and social media posts\n\n#### app/utils.py\n- Helper functions (logging, NLTK data download)\n- seed_demo_users() - Demo credentials setup\n\n### Frontend Files\n\n#### src/App.tsx\n- Main application component\n- Routing setup\n- Theme management\n\n#### src/pages/Auth.tsx\n- Login and registration forms\n- JWT token management\n\n#### src/contexts/AuthContext.tsx\n- Global authentication state\n- Token persistence\n- User session management\n\n#### src/lib/api.ts\n- API request helpers\n- Token management functions\n- Centralized API communication\n\n## Adding Features\n\n### Adding a New Backend Endpoint\n\n1. Create the endpoint in `app/routes.py`:\n```python\n@analysis_ns.route('/new-endpoint')\nclass NewEndpoint(Resource):\n    @api.expect(model)\n    def post(self):\n        \"\"\"Endpoint description\"\"\"\n        return {'result': 'data'}, 200\n```\n\n2. Update frontend API helper in `src/lib/api.ts`\n\n3. Create frontend page/component to use the endpoint\n\n### Adding a New Frontend Page\n\n1. Create page in `frontend/src/pages/NewPage.tsx`\n\n2. Add route in `src/App.tsx`:\n```jsx\nimport NewPage from './pages/NewPage';\n\n// In router config:\n{ path: '/new-page', element: <NewPage /> }\n```\n\n3. Add navigation link in layout\n\n## Database Migrations\n\nUsing Flask-Migrate:\n```powershell\n# Generate migration\nflask db migrate -m \"Description of changes\"\n\n# Apply migration\nflask db upgrade\n```\n\n## Testing\n\n### Run Tests\n```powershell\npytest tests/\n```\n\n### Test Coverage\n```powershell\ncoverage run -m pytest\ncoverage report\n```\n\n## Code Style\n\n### Python\n- Use Black for formatting: `black app/`\n- Use Flake8 for linting: `flake8 app/`\n\n### TypeScript/React\n- Use Prettier for formatting\n- ESLint configuration in `frontend/.eslintrc`\n\n## Environment Variables\n\nCreate a `.env` file in the root:\n```\nFLASK_ENV=development\nDEBUG=True\nJWT_SECRET_KEY=your-secret-key\nDATABASE_URL=sqlite:///dev.db\n```\n\n## Building for Production\n\n### Frontend Build\n```powershell\ncd frontend\nnpm run build\ncd ..\n```\n\n### Run with Gunicorn\n```powershell\ngunicorn -w 4 -b 0.0.0.0:5000 \"app:create_app()\"\n```\n\n## Common Issues and Solutions\n\n### CORS Errors\nCORS is enabled in Flask with Flask-CORS. If issues occur, check:\n- `app/__init__.py` CORS configuration\n- Frontend API URL matches backend\n\n### Token Expiration\nJWT tokens expire after 30 minutes by default. Check:\n- `app/config.py` JWT_ACCESS_TOKEN_EXPIRE_MINUTES\n- Frontend token refresh logic\n\n### Database Locked\nIf you get database locked errors:\n```powershell\nRemove-Item app\\dev.db\n```\n\n## Resources\n\n- Flask Documentation: https://flask.palletsprojects.com/\n- React Documentation: https://react.dev/\n- Vite Documentation: https://vitejs.dev/\n- SQLAlchemy: https://www.sqlalchemy.org/\n- Flask-RESTX: https://flask-restx.readthedocs.io/\n"
